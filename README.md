# CS-300-Analysis-and-Design
CS-300 Analysis and Design Repository
CS 300 Reflection: Data Structures and Algorithms
In this course, the main problem I was solving centered around how to efficiently organize, store, and retrieve course data using appropriate data structures. In Project One, I analyzed the run time and memory usage of different data structures to determine which approach would best support performance and scalability. In Project Two, I implemented working code that sorts and prints a list of Computer Science courses in alphanumeric order. Together, these projects focused on understanding how structure impacts both performance and usability in real applications.
My approach to solving these problems was grounded in selecting the right data structure for the job rather than simply making something work. This course reinforced that data structures are not just theoretical concepts but practical tools that directly affect efficiency and readability. By comparing options such as vectors, hash tables, and binary search trees, I was able to better understand how design decisions influence run time and memory consumption.
One of the biggest challenges I encountered was translating conceptual knowledge into working logic, particularly when implementing sorting and traversal in a way that maintained both accuracy and clarity. I worked through this by breaking the problem into smaller steps and validating each portion of the logic before moving forward. This helped me stay focused on functionality while still keeping long term maintainability in mind.
This project expanded how I think about software design by encouraging me to consider performance from the beginning instead of as an afterthought. I now approach development with a stronger awareness of how structure supports scalability and usability. Rather than focusing only on output, I place more emphasis on how data moves through a system and how future updates might impact the design.
As a result of this work, I also improved how I write programs that are maintainable, readable, and adaptable. I became more intentional about organizing logic clearly and selecting structures that support growth rather than limit it. This mindset will be especially valuable as I continue developing solutions that need to perform reliably over time.
